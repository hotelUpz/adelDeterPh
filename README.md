# Phemex Delisting Detector 🚀

**Phemex Delisting Detector** — это ультимативный, отказоустойчивый оркестратор для мониторинга биржи Phemex. Его главная цель — мгновенно обнаружить скрытый делистинг торговых пар (монет) и сверить их с вашими активными позициями, чтобы предупредить вас до того, как позиция будет принудительно закрыта биржей с потенциальным убытком.

## 🎯 Главная цель бота
Криптовалютные биржи (включая Phemex) часто проводят делистинг монет, принудительно закрывая открытые маржинальные позиции трейдеров по невыгодным ценам. 
Цель этого бота — **работать на опережение**. Бот непрерывно (раз в секунду) анализирует 3 независимых источника данных на предмет делистинга. Если монета попадает в "чёрный список", бот проверяет ваши открытые приватные позиции. Если вы торгуете этой монетой прямо сейчас — бот немедленно пробьёт тревогу через Telegram и Push-уведомления на телефон.

## ⚙️ Как работает пайплайн (Архитектура)

Гейм-лупа (основной цикл оркестратора) работает по следующему умному алгоритму, который экономит лимиты API биржи и гарантирует максимальную надежность:

### Шаг 1: Сбор кандидатов на делистинг (Public Sources)
Бот опрашивает три независимых источника:
1. **API Timeline (REST):** Проверяет внутреннее поле `timeline[3]` в публичном API биржи (анонс будущих изменений контракта).
2. **API Status (REST):** Проверяет поле `status` на наличие критических слов (например, `Delist`, `Settle`, `Offline`).
3. **Gmail Fallback (IMAP):** Проверяет вашу привязанную почту. Если биржа прислала письмо со словом `delist`, бот скачивает его батч-запросом и вычленяет из текста тикеры (вида `BTCUSDT`). *Опрос почты имеет внутренний троттлинг (по умолчанию раз в 5 секунд), чтобы Google не заблокировал доступ.*

### Шаг 2: Оптимизация
Если ни один из трёх источников не обнаружил ни одной монеты, готовящейся к делистингу — бот **засыпает на 1 секунду** и начинает цикл заново. *Он не делает лишних приватных запросов, экономя ваши rate limits.*

### Шаг 3: Проверка приватных позиций
Если обнаружен хотя бы один кандидат на делистинг, бот дергает приватный API Phemex (`/accounts/accountPositions`) используя ваши ключи, и собирает список всех монет, по которым у вас сейчас открыты позиции или ордера.

### Шаг 4: Пересечение и Триггер
Бот сравнивает два множества:
* `delisted_symbols` (монеты, которые биржа скоро удалит)
* `active_symbols` (ваши открытые позиции)

Если есть совпадение — бот фиксирует его, собирает информацию об источнике (кто именно из 3-х детекторов нашел сигнал) и отправляет экстренное уведомление.

### Шаг 5: Каскадное Уведомление
Сигнал передается в `NotificationManager`, который транслирует тревогу во все подключенные каналы:
* Telegram-бот (с указанием источника: `API Timeline`, `API Status` или `Email Fallback`)
* Pushover (Android)
* Alertzy (iOS)
* Techulus Push (iOS)

Бот будет повторять уведомления согласно настройкам (например, каждые 5 минут, до 5 раз), пока вы не закроете позицию. Как только вы закроете позицию на бирже, бот автоматически удалит монету из кэша тревог и успокоится.

## 🛠 Установка и Настройка

1. Создайте файл `.env` в корневой папке на основе шаблона и впишите свои ключи:
   * Токен Telegram бота (`TG_BOT_TOKEN`)
   * Ключи Phemex (`PHEMEX_API_KEY`, `PHEMEX_API_SECRET`)
   * Настройки Gmail (`GMAIL_USER`, `GMAIL_APP_PASSWORD`). Инструкция по получению пароля: см. `DOCS/GMAIL_SETUP.md`.
   * Токены Push-сервисов (по желанию).

2. Отредактируйте конфигурацию в `CONFIG/prod/` (лимиты, паузы, шаблоны сообщений).

3. Запуск: `python main.py`

## 🛡 Безопасность и SOLID
Бот написан с соблюдением принципов строгой изоляции состояния (SOLID). В нем отсутствуют глобальные переменные состояния, а все зависимости инжектируются в изолированный контекст. Парсинг почты и запросы к REST API полностью асинхронны (asyncio / aiohttp), что исключает блокировки гейм-лупы.
